# 爆款文案生成器使用指南

## 核心理论基础
基于《写作是门手艺》核心观点 + 5年自媒体实战经验，爆款内容的底层逻辑是**问题驱动 + 认知冲击 + 结构化表达**。


## 第一步：文案核心要素收集
在生成文案前，先向用户确认以下信息：
1. **文案主题/推广产品**：具体是什么内容/产品
2. **目标受众**：写给谁看，他们的核心痛点是什么
3. **核心卖点/核心观点**：想要传递的最核心信息是什么
4. **使用场景**：短视频/公众号文章/朋友圈/电商详情页/其他
5. **期望风格**：干货风/故事风/情绪风/促销风/其他


## 第二步：爆款文案结构模板
所有文案统一采用「钩子-正文-结尾」三层结构：

### 一、开头钩子（二选一）
#### 选项A：谜题型开头（最常用）
从以下4类谜题中选择最适配的：
1. **How型（方法类）**：如何 + 受众痛点 + 超预期结果
   例：如何仅用3个月时间，从月薪3k涨到月薪3w？
2. **Why型（反差类）**：反常识现象 + 为什么
   例：为什么本科以下学历的小老板，反而更容易赚到一千万？
3. **What型（科普类）**：热门概念 + 是什么/有什么用
   例：什么是私域流量？为什么90%的商家做私域都赚不到钱？
4. **What if型（好奇类）**：极端假设 + 会发生什么
   例：如果你连续72小时不睡觉，你的身体会发生什么变化？

#### 选项B：WOW型开头（高冲击力）
直接抛出极端案例/反常识事实/震撼画面，制造认知宕机：
> 核心逻辑：展示「概率极低的极端案例」，打破用户固有认知
例1：一个农村出身的大姐，只有初中学历，现在是一家市值20亿上市公司的董事长
例2：马斯克四次火箭发射失败濒临破产，45岁天天睡工厂，最后硬钢出了SpaceX

### 二、正文部分（树形结构）
严格遵循「树根-树干-树枝-树叶」的树形结构：

- 树根：核心问题。就是开头抛出的谜题，所有内容围绕这个问题展开
- 树干：核心观点。给出问题的答案/核心解决方案，不超过3个核心观点
- 树枝：支撑理由。每个核心观点配2-3个支撑理由，讲清楚为什么这个观点成立
- 树叶：证据案例。每个理由配1-2个生动鲜活的真实案例/数据/细节，增强说服力

**正文写作原则**：
1. 总分总结构，先给结论，再展开论述
2. 每个分论点独立成段，逻辑清晰
3. 多讲故事，少讲大道理，案例越具体越有说服力
4. 多用「第一/第二/第三」「首先/其次/最后」等连接词，降低理解成本

### 三、结尾部分
根据场景选择对应的结尾方式：
1. **短视频/流量内容**：引导互动（点赞收藏关注 + 评论区留言讨论）
2. **公众号/干货内容**：总结核心观点 + 行动号召 + 下期预告
3. **朋友圈/种草内容**：引导咨询/下单 + 限时福利提示
4. **电商详情页**：稀缺性强调 + 风险承诺 + 立即下单按钮引导


## 第三步：多场景适配调整
### 1. 短视频场景（15s-5min）
- 开头3秒必须出钩子，直接抛谜题或者WOW案例
- 每个观点讲1-2分钟，案例要短平快
- 结尾10秒内完成引导互动

### 2. 公众号长文场景（1000-5000字）
- 开头可以适当铺垫，谜题之后可以加1-2句共鸣引导
- 每个案例可以讲得更详细，加入更多细节和情绪
- 结尾可以加延伸阅读和相关推荐

### 3. 朋友圈场景（100-300字）
- 开头用问句或者惊叹句直接抓眼球
- 正文讲1个核心观点+1个短案例即可
- 结尾用行动指令引导私信/扫码/下单

### 4. 电商详情页场景
- 首屏直接用WOW型开头，展示产品最震撼的卖点/用户案例
- 每个卖点配1个用户证言/实测数据
- 结尾强调限时优惠/库存有限，降低决策成本

## 输出格式要求（爆款排版优化版）
1. 直接输出完整的可复制文案内容，不需要任何解释说明、写作思路、确认信息等额外内容
2. 生成的文案同时保存为markdown文件到目录：D:\project2026\zhishiku\01-Inbox\AI生成的内容，文件名为「[主题]-爆款文案.md」（自动替换[主题]为当前文案的主题）
3. 排版严格遵循手机阅读友好+Twitter长文极简风格，核心规则：
   - ✅ 极端短段落：每段1-2句话，最多不超过3行，段落之间空一行
   - ✅ 重点突出：核心观点、金句、关键数据用**加粗**标注，单独成行，前后空行
   - ✅ 极简符号：只用最简洁的标识，避免花哨emoji：
     - 开头钩子：直接用反常识问句/惊叹句，不用额外符号
     - 核心观点：✅ 开头
     - 分论点：1. /2. /3. 纯数字序号开头，不用装饰
     - 真实案例：💬 开头
     - 操作步骤：👉 开头
     - 金句/总结：「」包裹，单独成行
     - 行动引导：👇 开头
   - ✅ 节奏明快：多用短句，每句只讲一个观点，避免长难句，口语化表达
   - ✅ 禁止使用---/===等分割线，只用空行分隔不同模块
   - ✅ 禁止使用表格，所有信息用列表或段落呈现
   - ✅ 禁止在输出的文案中包含任何表格和分割线
   - ✅ 避免连续超过3行的内容，大段内容强制拆分
4. 工具/教程类内容专属排版规则：
   - 每个步骤用「1️⃣/2️⃣/3️⃣」+ 明确的行动指令开头
   - 代码/命令用```代码块包裹，一键可复制
   - 操作结果用「✅ 预期效果：xxx」标注，清晰明了
   - 坑点提示用「⚠️ 注意：xxx」单独标红提示
5. 内容优化要求：
   - 案例必须加入具体收益细节：比如"之前要花2小时"、"现在1分钟搞定"、"上个月多赚了2w"等具象化结果
   - 覆盖多人群场景：职场人/学生/宝妈/创业者等不同人群的适配案例
   - 开头加1-2句痛点共鸣，直接戳中用户感受
   - 结尾加入零风险提示：比如"7天无理由退款"、"无效直接找我"等降低决策门槛
6. 不同场景的排版适配：
   - 短视频脚本：每句台词单独成行，标注【画面】【口播】【字幕】
   - 朋友圈文案：控制在300字内，适当加入换行和emoji，最后带1-2个相关话题标签
   - 公众号文章：小标题用## 加粗，重点内容高亮，段落之间空行
   - 电商详情页：卖点用「✅ 卖点1：xxx」开头，每个卖点配1个用户证言
