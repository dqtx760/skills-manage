# png-xiaohongshu 技能

本技能旨在将结构化的文案内容，自动化地转换为一套符合小红书风格的PNG图片（一张封面图 + N张配图）。

## 核心工作流

1.  **内容解析 (Parse)**
    *   接收用户输入的完整文案。
    *   将文案智能分解为两个部分：
        1.  `cover_title`: 用于封面图的醒目标题。通常是文章的第一行或核心亮点。
        2.  `content_blocks`: 一个包含N个内容块的列表。每个块对应一张配图（例如，文案中的每个`###`标题和其下的内容可以被视为一个块）。
    *   确定需要生成的配图总数N。

2.  **设计规划 (Plan)**
    *   **必须**阅读并参考 `references/design-patterns.md` 文件来指导整个设计过程。
    *   **封面设计**: 根据参考文件中的“封面图设计”原则，为 `cover_title` 规划一个布局（例如，中心焦点式）。
    *   **配图设计**: 遍历`content_blocks`列表，为每一个内容块从参考文件的“常见版式”中选择一个最合适的布局（例如，为“Top 5”类型的内容选择“列表清单式”）。

3.  **素材与风格 (Style)**
    *   检查 `assets/` 目录中是否有可用的资源：
        *   `assets/backgrounds/`: 可选用的背景图片。
        *   `assets/icons/`: 可用于点缀的图标。
    *   如果没有提供配色方案，选择一个通用、和谐的配色（参考`design-patterns.md`中的建议）。
    *   选择清晰易读的字体。

4.  **图片生成 (Generate)**
    *   这是一个迭代过程。你需要调用一个**底层图片生成工具**（例如 `canva`, `svg-architect` 或其他可用的文生图工具）来完成实际的图片创建。
    *   **生成封面**: 调用工具，传入`cover_title`、规划好的布局和风格，生成 `cover.png`。
    *   **生成配图**: 循环N次，每次调用工具，传入一个`content_block`的内容、对应的布局和风格，依次生成 `image_01.png`, `image_02.png`, ... `image_N.png`。
    *   所有图片应保存到项目的临时输出目录中。

5.  **交付成果 (Deliver)**
    *   完成所有图片的生成后，向用户报告任务完成。
    *   提供一个包含所有已生成图片文件路径的列表。

## 示例

**用户输入**:
```
🔥我的周报生成器，一周Prompt写一次就够了

### 1. 收集信息
把所有周报需要的素材喂给GPT

### 2. 定义角色
让GPT扮演一个专业的周报作者

### 3. 输出周报
使用最终指令输出结构化周报
```

**AI执行流程**:
1.  **解析**:
    *   `cover_title`: "🔥我的周报生成器，一周Prompt写一次就够了"
    *   `content_blocks`: [`"### 1. 收集信息..."`, `"### 2. 定义角色..."`, `"### 3. 输出周报..."`]
    *   `N = 3`
2.  **规划**:
    *   阅读 `references/design-patterns.md`。
    *   封面采用“中心焦点式”。
    *   3个内容块均采用“单点放大式”版式，并使用数字序号 `01`, `02`, `03`。
3.  **生成**:
    *   调用图片生成工具，创建 `cover.png`。
    *   循环3次，创建 `image_01.png`, `image_02.png`, `image_03.png`。
4.  **交付**:
    *   “已为您生成4张图片。”
    *   返回路径列表：`["/path/to/cover.png", "/path/to/image_01.png", ...]`
