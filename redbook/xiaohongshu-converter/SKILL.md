# 小红书风格转换器

将普通文章/长文内容转换为符合小红书平台风格的笔记格式，保留核心信息的同时，改用小红书用户喜爱的语气、结构和表达方式。

## 核心转换原则

### 1. 标题
- 开头用emoji吸引注意力
- 加入悬念、痛点或价值承诺
- 控制在20字以内，最好15字左右
- 可以用问句引发好奇

**示例：**
❌ 原文标题：《如何提高工作效率的五个方法》
✅ 转换后：💼 打工多年才发现的效率秘密！

### 2. 开头
- 直接切入痛点，引发共鸣
- 用口语化表达，像和朋友聊天
- 第一人称"我"、"分享"、"亲测"增加信任感
- 避免客套和铺垫

**示例：**
❌ "大家好，今天我想和大家谈谈..."
✅ "打工人谁不想下班准时走啊！我试了一个月终于找到方法了..."

### 3. 正文结构
- 分点清晰，用emoji标注每个要点
- 短句为主，避免长段落
- 每段1-3句话，留白多，方便手机阅读
- 重点内容可以用表情符号突出
- 将专业术语转化为大白话

### 4. 语气风格
- 口语化，用小红书常用词汇："谁懂啊"、"真的绝"、"我悟了"、"亲测有效"、"挖到宝"、"干货"
- 适度使用感叹号，但不要过度
- 态度真诚，像朋友分享经验
- 避免官方、生硬、学术化的语言

### 5. 结尾
- 总结核心观点
- 引导互动：提问引发评论，比如"你们平时都是怎么做的？欢迎交流！"
- 添加相关话题标签

### 6. 标签
- 5-8个标签
- 1-2个大领域标签（#职场 #自我提升）
- 3-5个精准内容标签（#效率提升 #工作方法）
- 1个热门标签可选（#打工人）

## 格式细节

### 代码/命令格式说明

小红书不支持Markdown代码块，所以遇到命令、代码请直接用普通文本写出，不需要用```包裹。

**示例：**
❌ ```bash
npm install -g opencli
```
✅ npm install -g opencli

如果有多个命令，每行分开写清楚即可。

### 换行和分隔

小红书粘贴时会自动压缩连续换行，所以**不同大主题/段落之间必须用 `.` 分隔**，同一个主题内的多行内容之间不需要加分隔。

正确格式：
```
🌐 标题
谁懂啊！开头内容...
发现这个神器...
.
💡 第一个主题
这里是第一行
这里是第二行（同一主题内连续写，不需要加点）
.
💡 第二个主题
...
```

即：**不同主题之间** → 换行 → 单独一行写 `.` → 换行 → 开始下一个主题。**.的上下不需要额外空行**。

同一个主题内排版：
- **emoji标题单独占一行**，换行后再写内容
- 多条列表内容之间保持正常换行，不需要额外空行
- 同一个主题内不要有多余的空行，保持紧凑

这样粘贴到小红书之后排版清晰，不会全部连在一起，也不会有多余的空白。

## 转换步骤

1. **提炼核心**：先读懂原文，提取3-5个核心要点
2. **重构结构**：按照小红书格式重新组织
3. **调整语气**：将书面语改为口语化表达
4. **添加表情**：合理使用emoji增加可读性
5. **生成标签**：根据内容生成相关标签

- 例如标题"亲测有效的时间管理干货"保存为 `时间管理干货.md`

## 注意事项

- 保留原文的核心信息和观点，不要改变原意
- 如果原文已经很短，直接优化风格即可，不需要强行拆分
- 如果原文很长，要适当精简，抓住重点，小红书笔记不宜过长
- 尊重原文作者观点，只改风格不改内容
- 如果原文涉及敏感话题、广告、违规内容，直接拒绝转换
